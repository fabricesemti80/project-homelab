# Docker Deployment

This directory owns the Docker deployment for services that used to be declared as NixOS `oci-containers` on `trinity`.
It is intentionally plain Docker Compose so it can run on non-Nix machines.

## Layout

- `docker-compose.yml`: main Compose entrypoint. It includes each service-specific Compose file.
- `arcane/`, `beszel/`, `cloudflared/`, `omni/`, `traefik/`, `whoami/`: one directory per service group.
- `scripts/render-secrets.sh`: renders ignored runtime files from the repository root `.envrc`.
- `scripts/render-omni-envrc.sh`: creates a fresh Omni CA, server cert, GPG key, Dex client secret, and Dex password hash, then updates the root `.envrc` Omni block.
- `scripts/deploy.sh`: renders runtime files and deploys either locally or to a remote Docker server.
- `runtime/`: generated config and secret mounts consumed by Compose. Ignored by Git.
- `secrets/`: local source secret material such as generated Omni PEM/GPG/admin-password files. Ignored by Git.

## Required Local Secrets

The root `.envrc` is local and ignored. It must provide:

- `CLOUDFLARED_TUNNEL_TOKEN`
- `TRAEFIK_CLOUDFLARE_API_TOKEN`, `TRAEFIK_CLOUDFLARE_ZONE_ID`, `TRAEFIK_CLOUDFLARE_EMAIL`
- `ARCANE_ENCRYPTION_KEY`, `ARCANE_JWT_SECRET`
- Omni values: `OMNI_CA_CERT_PEM`, `OMNI_TLS_CERT_PEM`, `OMNI_TLS_KEY_PEM`, `OMNI_GPG_KEY_ASC`, `OMNI_DEX_CLIENT_SECRET`, `OMNI_DEX_PASSWORD_HASH`, `OMNI_USER_EMAIL`

To generate a new Omni bundle:

```sh
mise run docker:omni:envrc
```

The Omni admin password generated by that flow is stored locally at:

```text
infra/docker/secrets/omni/admin-password
```

## Deploy

Local Docker host:

```sh
mise run docker:deploy
```

Remote Docker host:

```sh
export HOMELAB_DOCKER_HOST=fs@10.0.40.61
export HOMELAB_DOCKER_REMOTE_DIR=/opt/project-homelab/infra/docker
mise run docker:deploy
```

The deploy script renders `runtime/` and `infra/docker/.env`, copies this Docker bundle to the remote host over SSH, then runs:

```sh
docker compose -f docker-compose.yml up -d --remove-orphans
```

## Beszel Agent

The Beszel agent is behind the `agent` Compose profile. Enable it only when `BESZEL_AGENT_KEY` and `BESZEL_AGENT_TOKEN` are set:

```sh
export COMPOSE_PROFILES=agent
mise run docker:deploy
```

## Omni Networking

Omni remains `network_mode: host` because SideroLink/WireGuard is sensitive to Docker bridge/NAT behavior. Traefik still provides normal public browser routing for:

- `https://omni.krapulax.dev/`
- `https://auth.krapulax.dev/`

Machine API, Kubernetes proxy, event sink, and WireGuard ports are still host-level ports and must be reachable by the Talos machines that need them.

## Rollback

From the Docker host directory:

```sh
docker compose -f docker-compose.yml down
```

Persistent state is in Docker named volumes prefixed with `homelab_`. Do not remove those volumes unless you intentionally want to erase service data.
